All New, Super Semantic Tags

HTML5 comes with a bunch of new block level elements for your semantic delight. Gone are the days when everything was a div.

Header

The header element groups together a header.

You can have as many header elements as you like on a page. If your page contains lots of logical sections, this might make sense. On most pages though you will only have one.

Footers

The footer element is similar, in that it groups together footer information.

Article

An article element represents a self contained article on a page. It represents a single, complete and coherent piece of text, what you or I might describe as an article.

"The article element represents a self-contained composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content." - says the W3C

Section

A Section represents a chunk of content with related meaning. Say you have an article about HTML5. You have some content about figures, some about headers, etc. Each part of the article might be a section. It's a sensible way to split up your content.

It's not supposed to be used just as a wrapper for styling though, it should be a real, meaningful section. Typically sections are found within articles, but not always. A section will typically have a heading at the top of it, but not always.

The section element represents a generic document or application section. The section element is not a generic container element. When an element is needed for styling purposes or as a convenience for scripting, authors are encouraged to use the div element instead. - the W3C

Navigation

The Nav tag wraps a series of navigation elements. Awesome.

Aside

The Aside element is used for content that is related, but tangental to the main content. For example text quotes, and navigation.

"The aside element represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.

The element can be used for typographical effects like pull quotes or sidebars, for advertising, for groups of nav elements, and for other content that is considered separate from the main content of the page." The W3C

This is a lovely kitten

Figure and Figcaption

These elements group together an image, or group of images with a caption. This is a very cool thing to be able to do.

The W3C says:

The figure element represents some flow content, optionally with a caption, that is self-contained and is typically referenced as a single unit from the main flow of the document.

The element can thus be used to annotate illustrations, diagrams, photos, code listings, etc, that are referred to from the main content of the document, but that could, without affecting the flow of the document, be moved away from that primary content, e.g. to the side of the page, to dedicated pages, or to an appendix.

Note that figures do not necessarily have to be images. A code listing, poem, or anything that illustrates a point, but is not the point, could be put in a figure. Also, you don't need to wrap every image in a figure. Use them for extra content that illustrates your point.

Hgroup

The hgroup tag represents a group of related header elements. Hgroups can only contain the tags h1 to h6.

Often, when we write a title, we want a subtitle as well. In the past, we've had two choices, either

  1. Write the subtitle as a paragraph, then style it large
  2. Write the subtitle as an h2, and mess up the document hierarchy.

With an hgroup, we can use an h2 without messing up the hierarchy.

Div

A div is an arbitrary document division. Use it to mark up any section of the page.

Span

A span is an inline section of the page, use it for marking up bits of text within the text flow.
